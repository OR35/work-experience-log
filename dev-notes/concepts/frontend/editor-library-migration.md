# [Concept] 사용자 경험 중심의 에디터 라이브러리 교체 전략 (Toast UI → Quill)

풍부한 텍스트 편집 경험과 세밀한 이미지 제어 기능을 제공하기 위해 에디터 엔진을 교체하고 기능을 확장한 전략입니다.

---

## 1. 배경 (Context)
- **라이브러리 제약:** 기존 Toast UI Editor는 Markdown 기반으로 안정적이나, WYSIWYG 모드에서 이미지 정렬(좌/우/중앙) 및 글꼴 크기/색상 등 세부 스타일 커스터마이징에 제약이 있음.
- **요구 사항:** 1. 이미지 클릭 시 개별 정렬 기능 제공. 2. 폰트 크기, 배경색 등 워드 프로세서 수준의 편집 기능 확보. 3. Vue 3 반응형 시스템과의 안정적인 데이터 동기화.

---

## 2. 해결 전략
- **엔진 교체 (Quill.js):** DOM 기반으로 동작하여 커스터마이징이 용이하고 확장성이 뛰어난 Quill을 선택.
- **이미지 정렬 알고리즘 구현:** - Quill의 기본 `align` 핸들러를 오버라이딩하여, 텍스트뿐만 아니라 선택된 이미지(`IMG` 태그)의 스타일(`margin`, `display`)을 직접 제어하도록 구현.
  - `Attributor`를 등록하여 인라인 스타일 기반의 정렬 상태가 HTML 소스에 정확히 반영되도록 설계.
- **상태 동기화 최적화:** `isUpdating` 플래그를 도입하여 `props` 변경으로 인한 루프(Infinite Loop)를 방지하고 데이터 무결성 확보.

---

## 3. 기대 효과
- **콘텐츠 품질 향상:** 사용자 의도에 맞는 유연한 레이아웃 구성 가능.
- **유지보수 용이성:** 순수 JavaScript 기반의 Quill 핸들러 제어를 통해 향후 추가 요구사항(이미지 리사이징 등)에 유연하게 대응 가능.
