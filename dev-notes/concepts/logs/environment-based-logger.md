# [Concept] 운영 환경 보안을 위한 환경별 로그 제어 전략

운영 환경에서 내부 데이터 노출을 방지하고, 개발 단계에서는 디버깅 효율을 극대화하기 위한 로그 관리 전략입니다.

---

## 1. 배경 (Context)
- **보안 리스크:** `console.log`에 담긴 API 응답값이나 비즈니스 로직 정보가 운영 환경 개발자 도구에 노출되는 문제.
- **개발자 경험(DX):** 수많은 로그 사이에서 필요한 정보를 빠르게 식별하고, 객체 데이터를 시각적으로 파악하기 어려움.

## 2. 해결 전략
- **로그 레벨 분리:** `log`, `info`, `debug`, `error`로 세분화하여 각 환경(`local`, `dev`, `prod`)에 맞는 노출 범위 설정.
- **전역 로거(Global Logger):** 매번 `import` 하는 번거로움을 없애고 `window` 객체를 통해 어디서든 접근 가능하도록 설계.
- **데이터 시각화:** `console.table` 및 `console.group`을 활용하여 복잡한 데이터 구조를 직관적으로 확인.

---

## 3. 기대 효과
- **운영 안정성:** 실제 배포 환경에서는 `error` 외의 로그를 원천 차단하여 정보 유출 방지.
- **디버깅 속도:** 로컬 환경에서 핵심 데이터를 테이블 형태로 확인하여 데이터 정제 시간 단축.
