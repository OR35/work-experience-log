# [Refactoring] 데이터 기반 동적 렌더링을 통한 메일 가이드 관리 체계 개선

## 1. 개요 (Context)
프로젝트 내 메일 발송 관리 시스템에서 각 메일 유형별로 사용할 수 있는 치환 키워드(가이드)를 사용자에게 제공하는 기능을 개선했습니다.
초기 구조는 모든 키워드가 HTML 내에 하드코딩되어 있어, 서비스가 확장됨에 따라 운영 효율성이 급격히 저하되는 구조였습니다.

---

## 2. 문제점 (Problem)
- **높은 수정 비용**: 가이드 내용이나 키워드 문구가 변경될 때마다 관련 `.vue` 파일의 마크업을 일일이 찾아 수정해야 하는 번거로움이 있었습니다.
- **데이터 중복**: 플랫폼명, URL 등 모든 메일에서 공통으로 쓰이는 키워드들이 각 유형별 컴포넌트에 중복으로 존재하여 정합성 관리가 어려웠습니다.
- **유연성 부족**: 메일 유형이 추가될 때마다 유사한 HTML 구조를 복사하여 붙여넣는 방식이 반복되어 코드의 비대화를 초래했습니다.

---

## 3. 해결 방안 (Action)

### UI와 데이터의 완전한 분리
하드코딩된 테이블 행(`TableRow`)을 제거하고, 모든 가이드 정보를 객체 배열 형태의 상수 데이터(`mailGuide.js`)로 외부화했습니다.

### 데이터 구조의 계층화
데이터를 **'공통 키워드(Common)'**와 **'유형별 키워드(Type-specific)'**로 계층화하여 관리 효율을 높였습니다.

- **공통**: 플랫폼 정보, 회원 기본 정보 등.
- **특화**: 가입(비밀번호), 데이터셋(데이터셋명), 최적화(수행시간) 등 유형에 따른 가변 데이터.

### 동적 매핑 로직 구현
Vue의 `computed`를 활용하여 현재 선택된 메일 유형 코드에 따라 공통 키워드와 특화 키워드를 실시간으로 병합하여 테이블에 주입하는 로직을 구현했습니다.

---

## 4. 리팩토링 성과 (Result)

### 개발 및 유지보수 효율성 증대
- **코드량 감소**: 기존 60라인 이상의 중복 HTML 코드를 10라인 내외의 동적 반복문(`v-for`)으로 축약했습니다.
- **운영 안정성**: 가이드 문구 수정 시 마크업 수정 없이 상수 파일(`mailGuide.js`) 한 곳만 업데이트하면 되므로 휴먼 에러를 방지했습니다.

### 비즈니스 확장성 확보
- 신규 메일 유형이 추가될 때 UI 개발 없이 상수 객체에 데이터만 추가하면 즉시 가이드 테이블이 자동 생성되는 구조를 확보했습니다.

---

## 5. 핵심 역량 (Key Skills)
- **Vue.js**: Computed를 활용한 반응형 데이터 처리 및 선언적 렌더링.
- **Architecture**: 데이터와 UI의 관심사 분리(Separation of Concerns).
- **Clean Code**: 중복 제거 및 유지보수가 용이한 상수 관리 체계 구축.
