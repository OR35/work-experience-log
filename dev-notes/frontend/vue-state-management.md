# Vue State Management Pattern

## 목적
Vue 기반 SPA에서 상태 관리 혼선을 줄이기 위한 기준 정리

---

## 상태 분리 기준

- 화면 상태 (UI)
- 데이터 상태 (API 결과)
- 제어 상태 (권한, 로딩, 에러)

---

## 자주 발생한 문제

- 데이터는 있는데 화면에 안 나옴
- 상태 변경 시점과 렌더링 불일치

---

## 해결 전략

- onMounted / watch 타이밍 명확히 구분
- 상태 준비 여부에 따른 조건 렌더링
- 공통 상태 초기화 로직 구성

---

## 배운 점

- 상태 문제는 대부분 타이밍 문제
- 무작정 watch 추가는 오히려 복잡도 증가
